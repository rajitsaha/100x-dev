# 100x-dev: Multi-Tool AI Development Workflows

**Date:** 2026-04-10
**Status:** Approved
**Scope:** Restructure and rebrand `claude-dev-setup` into `100x-dev`, a universal AI coding tool workflow system.

---

## Overview

Restructure the repository from a Claude Code-specific setup into a tool-agnostic workflow system that works with Claude Code, Cursor, Codex (OpenAI), Windsurf, Copilot CLI, Gemini CLI, and Antigravity. The core insight: the workflows (quality gates, test runners, security scans, etc.) are already tool-agnostic markdown + bash. Only the delivery mechanism is tool-specific.

No new workflows or functionality. This is a restructure + rebrand.

## Workflow File Changes

Minimal edits to make workflows tool-agnostic:

1. **Remove `$ARGUMENTS`** from end of files (Claude Code-specific). Each adapter handles argument passing its own way.
2. **Change cross-references**: "Invoke `/test`" → "Run the **test** workflow". Adapters map this back to tool-specific syntax if needed.
3. **Remove Claude Code assumptions** in prose (e.g., "Restart Claude Code to load").
4. **Keep all bash commands, thresholds, and logic exactly as-is.** The gate thresholds (95% coverage, zero critical vulns), detection logic, and opinionated philosophy are unchanged.

Example change in gate.md:
```
# Before
Invoke `/test`. Run ALL test layers...

# After
Run the **test** workflow. Run ALL test layers...
```

## Adapter Design

Each adapter is a shell script with two functions:

```bash
#!/usr/bin/env bash
# Adapter interface:
#   install_global <workflows_dir>                    — install globally (if tool supports it)
#   install_project <workflows_dir> <project_path>    — generate project instruction file
```

### Global install (tools that support it)

| Tool | Global location |
|------|----------------|
| Claude Code | `~/.claude/commands/*.md` (copy individual files) |

### Project install (single-file generation)

| Tool | Output file | Method |
|------|-------------|--------|
| Cursor | `.cursorrules` | Concatenate workflows with `---` separators |
| Codex | `AGENTS.md` | Concatenate workflows with `---` separators |
| Windsurf | `.windsurfrules` | Concatenate workflows with `---` separators |
| Copilot | `.github/copilot-instructions.md` | Concatenate workflows with `---` separators |
| Gemini CLI | `GEMINI.md` | Concatenate workflows with `---` separators |
| Antigravity | `ANTIGRAVITY.md` | Placeholder, update when format is known |

### Concatenation format for single-file tools

```markdown
# 100x Dev Workflows
# Generated by 100x-dev (https://github.com/rajitsaha/100x-dev)

---

# Gate — Pre-Commit Quality Gate

[contents of gate.md]

---

# Test — Run All Tests

[contents of test.md]

---
...
```

### Claude Code adapter (special)

In addition to copying workflow files, the Claude Code adapter also:
- Installs plugins from `plugins/plugins.json` into `~/.claude/settings.json`
- Appends `\n$ARGUMENTS` as the last line of each workflow file when copying to `~/.claude/commands/` (Claude Code substitutes this with user-provided arguments)

## Installer

`install.sh` is a multi-tool interactive installer with two selection steps:

### Step 1: Tool selection

```
Which AI coding tools do you use?
  [x] 1) Claude Code
  [ ] 2) Cursor
  [ ] 3) Codex (OpenAI)
  [ ] 4) Windsurf
  [ ] 5) Copilot CLI
  [ ] 6) Gemini CLI
  [ ] 7) Antigravity

Toggle (1-7) or press Enter to confirm:
```

### Step 2: Component selection

```
What would you like to install?
  [x] 1) Workflows     — /gate, /test, /commit, /push, /launch, ...
  [x] 2) Plugins       — Claude Code only: superpowers, stripe, hookify, ...
  [x] 3) Shell aliases — cc, ccc, 100x-update, 100x-check
  [x] 4) Templates     — project starters (node, python, docker)

Toggle (1-4) or press Enter to confirm:
```

Plugins option only shown if Claude Code is selected.

### Behavior per tool

- **Claude Code**: runs `adapters/claude-code.sh` global install (copies to `~/.claude/commands/` + optional plugins)
- **All other tools**: asks for project path (default: current directory), runs adapter to generate the instruction file there
